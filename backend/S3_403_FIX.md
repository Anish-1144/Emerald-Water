# Fix S3 403 Forbidden Error

If you're getting `403 Forbidden` when accessing S3 images, your bucket needs a public read policy.

## Quick Fix

1. **Go to AWS S3 Console** → Your bucket: `label-image-upload-canada`

2. **Click on Permissions tab**

3. **Scroll to "Block public access"** → Click **Edit**
   - **Uncheck** "Block all public access"
   - Click **Save changes**
   - Type `confirm` to confirm

4. **Scroll to "Bucket policy"** → Click **Edit**

5. **Paste this policy** (replace bucket name if different):

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "PublicReadGetObject",
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::label-image-upload-canada/*"
    }
  ]
}
```

6. **Click Save changes**

7. **Test** - Try accessing your image URL again:
   ```
   https://label-image-upload-canada.s3.ca-central-1.amazonaws.com/label-images/1770619255104-633b843c10364b53.png
   ```

## Alternative: Use CloudFront (Recommended for Production)

If you don't want public access, use CloudFront CDN:
- Better performance
- More secure
- Lower costs for high traffic
- Requires signed URLs or CloudFront origin access

## Verify It's Working

After setting the bucket policy, your images should be accessible via:
- Direct URL: `https://label-image-upload-canada.s3.ca-central-1.amazonaws.com/path/to/image.png`
- Should return 200 OK instead of 403 Forbidden
